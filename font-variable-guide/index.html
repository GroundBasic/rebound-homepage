1

위 줄 = 아웃라인, 아래 줄 = 필드 두 줄로 정렬되어 있어요.

🤔 핵심 질문 — Outline과 Filled를 어떻게 관리할까?

아이콘 하나당 아웃라인 버전과 필드 버전이 있어요.
이걸 두 개의 독립 컴포넌트로 만들 수도 있고,
하나의 컴포넌트 안에 Variant로 묶을 수도 있어요.

🔵 Phase 2 — 아이콘을 컴포넌트로 변환
1

안의 아이콘 프레임들을 드래그로 전체 선택하거나
카테고리 프레임 클릭 후 Cmd+A로 자식 전체 선택
2
컴포넌트로 변환
Cmd+Alt+K
선택된 모든 아이콘 프레임이 한 번에 컴포넌트로 바뀌어요!
Layers 패널에서 ◇ 보라색 다이아몬드로 변하면 성공
3
모든 카테고리 반복
Actions → Navigation → Settings → Media&Files → Alerts → Devices → Arrows
카테고리별로 같은 방법으로 반복해요.
⭐ Phase 3 — Outlined + Filled를 하나의 컴포넌트로 묶기
이제 같은 아이콘의 Outlined / Filled 두 컴포넌트를 하나로 합쳐서
Variant 속성(Style)을 만들어요.
1
같은 아이콘의 Outlined + Filled 두 컴포넌트 선택
예) search-outlined + search-filled 두 개를
Shift+클릭으로 함께 선택해요.
2
Combine as Variants
상단 툴바에서 Combine as variants 버튼 클릭
또는 우클릭 → Combine as variants
보라색 점선 테두리 안으로 두 컴포넌트가 묶여요!
3
Property 이름 정리
우측 패널 → Component properties에서
자동으로 생긴 Property 이름을 Property 1 → Style 로 변경
값은 search-outlined → Outlined, search-filled → Filled로 정리
4
컴포넌트 세트 이름 정하기
묶인 컴포넌트 세트(보라 점선 테두리)를 클릭
이름을 icon/search 로 입력
이제 Assets 패널에서 icon 폴더 안의 search로 보여요!
5
나머지 아이콘도 반복
favorite, home, arrow-back, add, settings 등
모든 아이콘에 대해 같은 과정을 반복해요.
Combine as Variants 결과
⬚icon/search(컴포넌트 세트)
├◇Style=Outlined
└◇Style=Filled
✅ 완성! 이제 버튼 안에서 icon/search 인스턴스를 넣고, 우측에서 Style: Outlined / Filled를 드롭다운으로 선택할 수 있어요.
4
아이콘 색상 변수 설정 + 연결
아이콘 색이 버튼 상태(기본/Hover/Disabled)에 따라 자동으로 바뀌려면
색상 변수를 아이콘 안의 텍스트 색상에 연결해야 해요.

🔗 아이콘 컴포넌트 안에 색 변수 연결
⚠️ Material Symbols는 폰트 기반!
일반 SVG 아이콘은 Fill(채우기)에 색이 있지만,
Material Symbols는 텍스트 레이어라서 텍스트 색상(Color)에 연결해야 해요.
1
아이콘 컴포넌트(Outlined 버전) 더블클릭
캔버스에서 컴포넌트를 더블클릭해서 내부로 진입해요.
Layers 패널에서 텍스트 레이어를 선택해요.
2
텍스트 색상에 변수 연결
우측 패널 → Fill (텍스트라서 Fill이 텍스트 색이에요)
색상 샘플 옆 ⊞ (Variable 버튼) 클릭
→ Semantic 컬렉션 → icon/color/on-dark 선택
3
Filled 버전도 동일하게 적용
Filled Variant로 이동 (Layers에서 Style=Filled 선택)
같은 방법으로 텍스트 색 → icon/color/on-dark 연결
4
모든 아이콘 컴포넌트 반복
검색, 홈, 즐겨찾기 등 모든 아이콘 컴포넌트에 동일하게 적용해요.
🎯 색 변수 연동 흐름
버튼 기본상태 (검정 채운 버튼)
버튼 배경색 ← btn/bg/black/default
아이콘 색 ← icon/color/on-dark → #FFFFFF
텍스트 색 ← btn/text/on-dark → #FFFFFF
Disabled 상태로 Variant 바뀌면 자동으로 →
버튼 배경색 ← btn/bg/black/disabled → gray/200
아이콘 색 ← icon/color/disabled → gray/400
텍스트 색 ← btn/text/disabled → gray/400
✅ Variant를 기본상태 → Disabled로만 바꾸면 배경색, 아이콘 색, 텍스트 색이 모두 한 번에 자동으로 바뀌어요!
5
Boolean 프로퍼티 연결 — 아이콘 보이기/숨기기
지금 버튼에 아이콘이 들어간 상태죠?
이제 Boolean 프로퍼티를 연결해서 아이콘을 토글로 켜고 끌 수 있게 만들어요.
💡 Boolean 프로퍼티가 뭔가요?
❌ 없으면
아이콘 없는 버튼 만들려면
직접 레이어 숨김 처리
인스턴스마다 일일이 수동 😩
✅ 있으면
우측 패널에서
토글 스위치 한 번으로
아이콘 On/Off 전환 🎉
🔧 Boolean 연결 방법 — 왼쪽 아이콘
1
버튼 컴포넌트 더블클릭 → 내부 진입
버튼 컴포넌트를 더블클릭해서 편집 모드로 들어가요.
⚠️ 인스턴스가 아닌 원본 컴포넌트를 더블클릭해야 해요!
2
왼쪽 아이콘 레이어 선택
Layers 패널 또는 캔버스에서 왼쪽 아이콘 인스턴스 클릭해서 선택해요.
3
눈(Visibility) 옆 ⊞ 클릭
우측 Design 패널 맨 위 레이어 영역을 보면
레이어 이름 옆에 작은 ⊞ 아이콘이 있어요.
(없으면 레이어 이름에 마우스 호버하면 나타나요)
4
"Boolean" 타입 선택 → 이름 입력
팝업에서 타입: Boolean 선택
이름: Left Icon (또는 왼쪽 아이콘)
기본값: True (켜진 상태)
Create property 클릭!
5
오른쪽 아이콘도 동일하게 반복
오른쪽 아이콘 인스턴스 선택 → 같은 방법으로
이름: Right Icon, 타입: Boolean, 기본값: True
✅ 완료! 이제 버튼 인스턴스를 선택하면 우측 패널에 Left Icon ✓ / Right Icon ✓ 토글이 보여요. 체크 해제하면 아이콘이 사라져요!

🗂️ 어느 컬렉션에 넣어야 해?
❌
Primitive 컬렉션에 넣기
Primitive는 원재료 보관소예요. danger/bg 같은 역할 변수를 여기 넣으면 섞여서 나중에 찾기 힘들어요.
✅
Semantic 컬렉션 새로 만들기 (권장)
Variables 패널 → + 컬렉션 추가 → 이름: Semantic
여기에 danger/bg, warning/border 등 역할 변수를 넣고 Primitive 값을 참조하게 해요.

💡 이렇게 하면 뭐가 좋아?
✓ 브랜드 색이 바뀌어도 Primitive의 danger/50 값만 바꾸면 → 연결된 모든 컴포넌트 자동 변경
✓ 협업 시 개발자에게 "danger/bg 써주세요" 라고 말하면 됨 → 코드에서도 같은 토큰명 사용 가능
✓ 다크모드 추가할 때 Semantic 변수에 Dark 모드만 추가하면 끝 → 컴포넌트 건드릴 필요 없음
