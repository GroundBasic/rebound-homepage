Step 1
컬렉션 & 그룹 구조 설계
Figma Variables는 슬래시(/)로 폴더 구조를 만들 수 있어요. 나중에 바꾸기 어려우니 처음에 구조를 잘 잡는 게 중요합니다.
Collection 1

01 Primitives

모든 컬러의 원시값. 절대 컴포넌트에 직접 연결하지 않습니다. Semantic이 이 값을 참조합니다.

  • 실제 hex 값이 들어가는 곳
  • Orange · Green · Blue Gray · Gray 4가지 스케일
  • 스케일 전체(0~100 또는 50~900)를 빠짐없이 등록
  • 이름 예: orange/50, gray/900
Collection 2

02 Semantic

용도 기반 이름. Primitive 값을 참조(alias)합니다. 컴포넌트는 여기에만 연결합니다.

  • hex 없음 — Primitive 토큰을 참조
  • Text · Background · Border · Icon · Interactive · Status
  • 이름 예: text/primary, interactive/primary/hover
  • 나중에 브랜드 컬러 바꿀 때 Primitive 1개만 수정
💡
핵심 원칙: 컴포넌트 Fill에는 #FF5900을 절대 직접 쓰지 않습니다. 반드시 interactive/primary/default 같은 Semantic 토큰을 연결하세요. 그래야 브랜드 컬러가 바뀔 때 Primitive 하나만 수정하면 전체가 자동으로 업데이트됩니다.

Step 4
Figma에서 실제로 설정하는 방법
Figma UI를 직접 조작하는 순서입니다. 처음에 구조가 중요하니 순서대로 따라하세요.
1

Variables 패널 열기

우측 패널 상단 또는 상단 메뉴에서 접근합니다.

  1. 우측 패널 상단 → Local variables 버튼 클릭
  2. 또는 메뉴 Edit → Manage local variables
2

01 Primitives 컬렉션 만들기

먼저 원시값 컬렉션을 만들고, 그룹별로 변수를 추가합니다.

  1. Variables 패널 좌측 상단 + → Create collection
  2. 컬렉션 이름: 01 Primitives 입력 후 Enter
  3. + Add variable → Color 선택
  4. 이름 입력: orange/50 → Enter → 색상 칩 클릭 → #FF5900 입력
  5. 이름에 /를 쓰면 자동으로 그룹(폴더)이 생깁니다
  6. 위 목록의 모든 변수를 순서대로 추가합니다
⚡
한 그룹 안에서 변수를 추가할 때 이전 변수를 선택한 상태에서 Enter를 누르면 같은 그룹에 빠르게 추가됩니다.
3

02 Semantic 컬렉션 만들고 참조(Alias) 연결하기

Semantic은 hex 값 대신 Primitive 변수를 참조합니다. 이게 핵심입니다.

  1. 다시 + → Create collection → 이름: 02 Semantic
  2. + Add variable → Color
  3. 이름: text/title 입력
  4. 값 영역의 색상 칩 클릭 → 색상 피커 오른쪽 상단 ⊞ 라이브러리 아이콘 클릭
  5. 검색창에 gray/900 입력 → 선택 → 참조 연결 완료
  6. 모든 Semantic 변수를 같은 방식으로 추가합니다
ℹ️
참조가 연결된 변수는 값 옆에 보라색 링크 아이콘이 표시됩니다. 이게 보이면 올바르게 연결된 것입니다.
4

컴포넌트에 Semantic 변수 연결하기

이제 실제 레이어의 색상을 hex에서 토큰으로 교체합니다.

  1. 버튼 레이어 선택 → 우측 패널 Fill 섹션의 색상 칩 클릭
  2. 색상 피커 우상단 ⊞ 아이콘 클릭
  3. interactive/primary/default 검색 후 선택
  4. Stroke, 텍스트도 동일하게 교체
  5. Prototype 없이도 Hover·Press 상태는 Interactive Components로 구현
⚠️
한 번에 다 바꾸지 않아도 됩니다. 새로 만드는 컴포넌트부터 Variables를 쓰고, 기존 것은 리팩토링하면서 점진적으로 교체하는 게 현실적입니다.
5

Library로 발행하고 팀과 공유하기

Variables를 다른 파일에서도 쓸 수 있도록 배포합니다.

  1. 좌측 Assets 탭 → Publish styles and variables 클릭
  2. 변경 내용 확인 후 Publish
  3. 팀원 파일 → Assets → Libraries → 이 파일을 Enable
  4. 이후 팀원도 Variables 패널에서 동일한 토큰 사용 가능
⚠️
Library 공유는 Figma Professional 이상 플랜에서 가능합니다. Starter 플랜은 같은 파일 내에서만 사용 가능합니다.

Step 6
Typography — Text Style + Number Variables
웹(64px)과 모바일(36px)처럼 플랫폼별로 크기가 달라지는 문제는 Number Variables의 Modes로 해결합니다. Text Style은 굵기·행간·자간 등 고정 속성만 담고, 크기는 Variable이 자동으로 바꿔줍니다.
Number Variable + Modes

폰트 크기 — 플랫폼별로 자동 변경

하나의 변수에 Web / Mobile 두 개 값을 넣어두면, 프레임의 Mode를 바꾸는 것만으로 안에 있는 모든 텍스트 크기가 한 번에 바뀝니다.

Text Style만 쓰는 경우

Text Style — 고정 속성만 담기

굵기(Bold/Regular), 행간(125%/140%/180%), 자간(-2.5%)처럼 플랫폼이 달라도 바뀌지 않는 값은 Text Style에 넣습니다.

  • Font weight → Text Style ✅
  • Line height → Text Style ✅
  • Letter spacing → Text Style ✅
  • Font size → Number Variable ✅
  • Color → Color Variable ✅
💡
핵심 워크플로우: 웹 프레임은 Mode = "Web", 앱 프레임은 Mode = "Mobile"로 설정. 그러면 같은 Text Style + Variable을 써도 각 프레임에서 크기가 자동으로 달라집니다. 나중에 Headline L 크기를 바꾸고 싶으면 Variable 하나만 수정하면 웹/모바일 모두 반영됩니다.

🛠 실제 설정 순서

1

03 Typography 컬렉션 만들기 + Modes 추가

  1. Variables 패널 → + Create collection → 이름: 03 Typography
  2. 우측 상단 Edit modes 클릭 → Mode 이름을 Web / Mobile 으로 변경
  3. + Add variable → Number
  4. 이름: font/headline-l → Web값: 64, Mobile값: 36 입력
  5. 위 목록의 모든 폰트 사이즈 변수를 추가
⚠️
Modes는 Figma Professional 이상 플랜에서 사용 가능합니다. Starter 플랜은 Mode가 1개(Default)만 지원돼서 이 기능을 쓸 수 없어요.
2

Text Style 만들기

  1. 임시 텍스트 레이어 만들고 → Font: Pretendard, Weight: Bold, Line height: 125%, Letter spacing: -2.5% 설정
  2. 우측 패널 Text 섹션 → 4-dot 아이콘(⊞) 클릭 → + Create Style
  3. 이름: Headline/L 입력 → 저장
  4. 나머지 스타일 반복 생성
💡
Text Style 이름에 슬래시(/)를 쓰면 Figma에서 자동으로 폴더로 그룹화됩니다. Headline/L, Headline/M은 Headline 폴더 안에 묶여요.
3

텍스트 레이어에 Style + Variable 동시 적용

  1. 텍스트 레이어 선택 → 우측 Text 섹션에서 Headline/L 스타일 적용
  2. Font size 숫자 칸 우클릭 → "Apply variable" → font/headline-l 선택
  3. 텍스트 Fill 색상 → Variable text/title 연결

이제 이 텍스트는 프레임의 Mode가 바뀌면 크기가 자동으로 변경됩니다.

4

웹 프레임 / 앱 프레임에서 Mode 전환

  1. 웹 디자인 최상위 Frame 선택 → 우측 패널 Variables 섹션 → 03 Typography → Web 선택
  2. 앱 디자인 최상위 Frame 선택 → 같은 방식으로 Mobile 선택
  3. 이제 두 프레임 안의 모든 텍스트 크기가 자동으로 달라집니다
🖥 Web Frame (Mode: Web)

📱 App Frame (Mode: Mobile)
